# 审核五关 · 检查项清单

> 本文件是 doc-review 技能的知识层。SKILL.md 只定义流程与输出格式，具体"查什么、怎么判"见本文件。
> 更新检查项时只改本文件，不动 SKILL.md。

---

## 第一关：文字差错关（基础，零容忍）

| 检查项 | 说明 | 语文教师高频示例 |
|--------|------|------------------|
| 错字 | 形近、音近、输入错误 | "即"与"既"、"己"与"已"与"巳"、"赢"与"羸" |
| 的/地/得 | 结构助词误用 | "认真的学习"✗→"认真的学习（名词前）"；"认真地学"✓；"学得好"✓ |
| 标点规范 | 书名号配对、引号嵌套、顿号逗号区分、问号滥用 | 概数"三四天"不加顿号；《〈论语〉十二章》内层用〈〉 |
| 异形词 | 推荐词形 vs 非推荐 | "唯一"而非"惟一"、"标志"而非"标识"、"订单"而非"定单" |
| 繁简混用 | 同一文档繁简一致 | "后"与"後"不混 |
| 数字用法 | 阿拉伯 vs 汉字 | 统计数字用阿拉伯，概数/固定表述用汉字（"十三五"不写"13·5"） |
| 专名错误 | 人名、篇名、书名、地名 | 作者与篇目错配、书名号误用 |

## 第二关：语法硬伤关（结构，零语病）

| 检查项 | 说明 | 语文教师高频坑 |
|--------|------|----------------|
| 成分残缺 | 主谓宾不完整 | "通过这次教研，使我的认识提高了"——"通过…使…"淹没了主语，删其一 |
| 搭配不当 | 主谓/动宾/修饰 | "提高…质量"✓ vs "提高…水平"✓；"培养…习惯"✓ vs "培养…意识"✓（莫乱配） |
| 语序不当 | 词语/分句顺序 | "许多附近的居民"→"附近的许多居民" |
| 句式杂糅 | 两式混用 | "原因是…造成的"→"原因是…"或"是…造成的" |
| 表意不明 | 指代/歧义 | "他告诉他自己错了"（两个"他"指代不清） |
| 不合逻辑 | 前后自洽 | "他是众多死难者中幸免的一个"→"遇难者中幸免的"；"大约三十岁左右"→"大约三十岁" |
| 否定失当 | 多否定翻转语义 | "谁也不能否认这出戏没有教育意义"→"谁也不能否认这出戏有教育意义" |
| 两面对一面 | 呼应不周 | "能否提高学生的语文素养，关键在于重视阅读"→前后加"能否/是否" |

## 第三关：用词精准关（核心，本技能的灵魂）

不只"没错"，更要"最准"。语文教师的文字，用词要经得起推敲：

| 检查项 | 说明 | 示例 |
|--------|------|------|
| 近义辨析 | 易混近义词选最贴切者 | "需要"vs"须要"（"须要"含"必须"义）；"做"vs"作"（"写作/作成"多用"作"，"做事/做饭"用"做"）；"启用"vs"起用" |
| 词语搭配 | 固定/习惯搭配 | "汲取…营养"✓；"吸取…教训"✓；莫"汲取教训""吸取营养" |
| 语义重复 | 堆砌同义 | "涉及到"→"涉及"；"凯旋归来"→"凯旋"；"亲眼目睹"→"目睹"；"悬殊很大"→"悬殊" |
| 褒贬色彩 | 感情色彩得体 | "趋之若鹜"（贬）莫用于褒义；"殚精竭虑"（褒）/"处心积虑"（贬）分清 |
| 语体得体 | 口语/书面/公文分寸 | 教案用教学语体；申报书用学术语体；推文用亲切语体；不混 |
| 成语误用 | 望文生义/对象错 | "万人空巷"（形容庆祝盛况，非"无人"）；"莘莘学子"（本身表复数，不加"众多"） |
| 生造词语 | 不规范新词 | 避免自造未被收录的词，确需用加引号并说明 |

## 第四关：文风与教学规范关（专业，体现语文素养）

| 检查项 | 说明 | 示例 |
|--------|------|------|
| 术语一致 | 同一概念全文统一 | 全篇统一用"学习任务群"或"任务群"，不中途换词 |
| 引用规范 | 课标/古文出处/教材版本 | 引《义务教育语文课程标准（2022 年版）》须全称；引古文注明篇目与所据版本 |
| 知识硬伤 | 文学/文化常识无误 | 作者朝代、篇目出处、名句归属不错（详见"古文引用专项审核"） |
| 去空话套话 | 剔除无信息量表述 | "在今后的工作中，我将继续努力……"如无具体，建议删或落地 |
| 语体统一 | 不忽正式忽口语 | 同一文体保持语体一致 |

## 第五关：格式审查关（特殊渠道格式合规）

语文教师的写作产物，常常投给特定渠道——征文比赛、期刊投稿、课题申报系统、单位公文、职称材料……这些渠道几乎都有硬性格式要求，且各不相同。内容再好，格式踩红线也可能被直接刷掉。

### 预检：主动问清"有没有特殊格式要求"（本关第一步，不可跳过）

- 审核任何文档时，先提醒/询问用户：这份文档是投给哪里的？有没有明确的格式规范（征文通知、投稿须知、申报书模板、单位公文模板、比赛要求等）？
- 若用户提供了格式规范（通知截图、模板文件、投稿须知文本）→ 提取其中的关键格式条款，逐条核对（见下方清单）。
- 若用户未提供 → 在报告中置顶提醒以下事项，并给出通用兜底建议，但不武断判定：
  > ⚠️ 未提供格式规范，无法做格式合规核对。如该文档有投稿/申报/参赛/上报等特定去向，请提供对应格式要求，否则可能踩格式红线被刷。
  > 通用兜底建议（无特殊要求时参考）：标题层级清晰、正文宋体/楷体小四至四号、1.5 倍行距、首行缩进两字符、字数自测并标注、文件名避免暴露无关信息。

### 格式审查清单（按渠道通用条目，据实勾选）

| 审查项 | 典型要求 | 核对要点 |
|--------|----------|----------|
| 标题格式 | 字号/字体/居中 | 黑体二号？是否需副标题？层级样式统一？ |
| 正文格式 | 字体/字号/行距 | 宋体四号？1.5 倍行距？首行缩进两字符？ |
| 字数要求 | 明确上限或区间 | "2000字左右""不超过5000字"——实测并标注是否达标 |
| 图文限制 | 是否禁图/需图 | 如"不含图片照片"须零图；或要求配图 |
| 空行/空段落 | 多余换行/空段 | 统计空段落数（段落文本去空白为空）；正文段落间不应有连续空行，空段落占比异常（如 >10%）应判 ❌/🟡 |
| 署名与单位 | 是否禁署名/署名位置 | 匿名评审、征文防识别须无姓名单位；署名在页眉/文末/系统表单？ |
| 结构要素 | 摘要/关键词/参考文献/页眉页脚 | 渠道是否强制要求这些模块 |
| 文件格式 | Word/PDF、版本、文件名 | 文件名是否含姓名（有的要求匿名，有的要求带姓名） |
| 合规附件 | AIGC 检测/查重/原创声明 | 如成人组须交 AIGC 报告且≤15%、查重率上限等 |

### 判定规则

- 清单每项给出 ✅合规 / ❌违规 / ⚠️未提供规范无法判定 三态之一。
- 违规项（❌）列入报告"📐 格式问题"小节，与内容问题（🔴🟡）分开列。
- 空行核查方法（硬性）：用 python-docx 读取段落，`空段落 = [p for p in doc.paragraphs if not p.text.strip()]`；报告空段落数与占比（空段落/总段落）。允许标题与正文、小节标题之间 1 个空段作间隔；若出现连续空段或空段落占比 >10%，判 🟡（建议删）乃至 ❌（严重空行泛滥）。Markdown 源稿的段间空行不应直接渲染成 Word 空段落——这是 AI/脚本生成文档的常见缺陷，必须主动查。
- 预检未提供规范时，报告置顶 ⚠️ 提醒，不生成 ❌（无据不判）。

## 古文引用专项审核（轻量子能力）

语文教师的写作常引古文，引错比现代文错更损专业形象。本技能不自建词汇大表（那归 `guwen-cidian`），只做"审核流程 + 判定规则"，数据回查 `guwen-cidian`。

### 审核流程

1. 提取文中所有古籍引用（篇目、作者、词句、所注版本）。
2. 核对作者与篇目归属（防"先天下之忧而忧——李白"这类硬伤）。
3. 核对词句原文（防张冠李戴、默写式错字、断句错误）。
4. 如需确认所据版本（通假字保留、异文、注音），回查 `guwen-cidian/references/教材版本/`。
5. 发现版本差异 → 标注说明，建议以统编教材为准。

### 判定依据（数据在 guwen-cidian）

- 通假/虚实词/活用/古今异义 → `guwen-cidian/references/古诗文基础/`
- 统编教材版本与差异 → `guwen-cidian/references/教材版本/`
- 中考高频词与真题例句 → `guwen-cidian/references/文言词表/`

### 降级策略

若 `guwen-cidian` 未安装：本关仍可基于通用文学常识做作者/篇目归属核对（防硬伤），但字义/版本/异文核对标注为 ⚪（需人工核实），不做强判定。

> 本技能只做"引用是否准确"的判定；具体字义讲解请直接调用 `guwen-cidian`。
