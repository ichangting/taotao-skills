# 语文写作审核（doc-review）—— 资深编审级咬文嚼字

## 技能定位

模拟资深语文编审的把关流程，为一线语文教师的写作产出做高精度审核。核心诉求：咬文嚼字，用词极准，不出现任何语法、词汇、用词方面的差错。

---

## 适用场景

| 写作产出 | 典型审核重点 |
|----------|--------------|
| 教案 / 教学设计 | 环节用语准确、指令清晰、不出现知识性硬伤、引用课文/课标无误 |
| 教学论文 / 教育随笔 | 论点表述严谨、文献与引文规范、无语病与逻辑漏洞 |
| 课题申报书 | 术语一致、研究内容表述精准、避免空话套话 |
| 公众号推文 / 短视频文案 | 面向家长学生，语体得体、通俗而不失准确、无硬伤 |
| 课件文本（PPT 文字） | 短句干净、重点词醒目、不与板书/口误混淆 |
| 学生范文评语 | 用词分量恰当（褒贬、激励尺度）、不出知识错误 |
| 考试命题语言 | 题干无歧义、选项无语病、指令唯一明确 |
| 教研发言稿 / 汇报 | 口语书面化转换、重点突出、用语规范 |

## 审核流程

### 第 0 步：确定审核模式

| 模式 | 覆盖关卡 | 适用 |
|------|----------|------|
| 快速校对 | 第一关（文字） | 即时通讯、朋友圈、紧急课件文字 |
| 标准审核 | 第一 + 二 + 三关 | 教案、公众号推文、学生评语、一般文章 |
| 深度审核 | 第一 + 二 + 三 + 四 + 五关 | 教学论文、课题申报书、考试命题、对外发布、投稿/参赛/申报 |
| 古文专项 | 古文引用专项审核 | 文中含较多古文引用时单独或叠加使用 |

未指定模式时，默认按"标准审核"执行；若文档用途为投稿/参赛/申报/对外发布，自动升级为"深度审核"。

### 第 1 步：读取检查清单

**读取 `references/审核清单.md`**，获取五关的具体检查项、示例与判定规则。未读检查清单就出报告属于严重失误。

### 第 2 步：逐关审核

按审核模式覆盖的关卡，逐项对照检查清单执行。每发现一处问题，记录：位置、原文、问题类型、修改建议、依据。

### 第 3 步：古文引用审核（如适用）

文中含古籍引用时，按 `references/审核清单.md` 中"古文引用专项审核"流程执行。数据回查 `guwen-cidian`；若 guwen-cidian 未安装，按降级策略处理（通用常识判硬伤，字义/版本标 ⚪）。

### 第 4 步：格式审查（深度审核时）

按 `references/审核清单.md` 中"第五关"流程：先问清格式要求 → 逐条核对 → 输出三态判定。

### 第 5 步：输出审核报告

按下方锁死格式输出。

## 审核报告模板（锁死格式，可复现）

```markdown
# 写作审核报告

**文档类型**：[教案/论文/课题/推文/课件/评语/命题/发言稿]
**审核模式**：[快速校对/标准审核/深度审核/古文专项]
**审核时间**：YYYY-MM-DD
**问题总计**：🔴 X 处 | 🟡 X 处 | 🟢 X 处 | ⚪ X 处

---

## 🔴 严重问题（必须改）
| 序号 | 位置 | 原文 | 问题类型 | 修改建议 | 依据 |
|------|------|------|----------|----------|------|

## 🟡 中等问题（建议改）
| 序号 | 位置 | 原文 | 问题类型 | 修改建议 |
|------|------|------|----------|----------|

## 🟢 轻微问题（可选优化）
| 序号 | 位置 | 原文 | 问题类型 | 优化建议 |
|------|------|------|----------|----------|

## ⚪ 存疑内容（需人工核实）
| 序号 | 位置 | 内容 | 疑点 |
|------|------|------|------|

## ✅ 已确认无误（抽查）
- [列出已核实正确的关键表述，增强作者信心]

## 📐 格式审查（深度审核时输出）
> 预检结论：[用户提供格式规范 / 未提供 —— ⚠️置顶提醒]
| 审查项 | 要求 | 状态 | 说明 |
|--------|------|------|------|

## 📝 整体建议
1. [结构/逻辑建议]
2. [文风建议]
3. [其他注意事项]
```

## 审核原则（铁律）

1. **零容忍硬伤** —— 错别字、语病、知识性错误，必须指出，不"差不多"。
2. **不确定标存疑** —— 无法核实的内容标 ⚪，不武断判定。
3. **修改给依据** —— 指出错误时尽量说明依据（语法规则/词典/教材版本），不只说"不对"。
4. **区分错误与建议** —— 事实错误/语病 ≠ 风格偏好，报告中分开列。
5. **保护原文意图** —— 只改错与优化表达，不替作者改观点、不改原有结构（除非作者要求）。
6. **尊重作者语体** —— 教案不必写成论文腔；推文不必端着。语体得体本身是一关。

---

## 限制说明

1. 本技能基于通用语文规范与 `guwen-cidian` 的文言语料，极冷门文献可能无法核实。
2. 学科专用术语（如跨到其他学科）需结合专业规范。
3. 学术争议问题无定论时，注明存在不同观点，不强行统一。
4. 审核是辅助，最终把关责任在作者本人。
