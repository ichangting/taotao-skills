# Changelog — doc-review

## [3.2.0] - 2026-07-21

### Changed
- **结构重构**：五关检查项知识层抽离至 `references/审核清单.md`，SKILL.md 只保留流程骨架与输出格式，token 消耗降低约 45%。
- **新增降级策略**：`guwen-cidian` 未安装时，古文引用审核基于通用常识判硬伤，字义/版本标 ⚪，不再静默失败。
- **新增 `data-freshness` 字段**：frontmatter 标注数据最后验证时间。
- 审核流程改为显式步骤（第 0-5 步），新增"第 1 步：读取检查清单"强制指令。

## [3.1.1] - 2026-07

### Added
- 格式审查清单新增「空行/空段落」审查项。
- 判定规则：用 python-docx 统计空段落数与占比；连续空段或占比 >10% 判 🟡/❌。
- 补强"Markdown 源稿段间空行被误渲染成 Word 空段落"的识别。

## [3.1.0] - 2026-06

### Added
- 新增「第五关：格式审查关」。
- 审核任何文档时先主动询问/提醒用户是否有特殊格式要求。
- 报告模板新增「📐 格式审查」小节，与内容问题分离列示。

## [3.0.0] - 2026-04

### Changed
- 从原"资深编审文档审查"五层体系重构，聚焦语文教师写作场景。
