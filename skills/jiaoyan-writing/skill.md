## 内置工具能力

### 一、PDF 解析能力（pdfplumber）

**已安装库**：pdfplumber 0.11.9

**支持功能**：
| 功能 | 说明 | 适用场景 |
|------|------|----------|
| 文字提取 | 提取 PDF 中的文字内容 | 电子 PDF |
| 表格提取 | 提取 PDF 中的表格数据 | 含表格的 PDF |
| 页面分析 | 分析页面结构和布局 | 复杂排版 |
| 元数据读取 | 读取 PDF 作者、标题等信息 | 参考文献整理 |

**技术实现**：
```python
import pdfplumber

def parse_pdf(pdf_path):
    """提取 PDF 内容"""
    result = {
        'text': '',
        'tables': [],
        'metadata': {}
    }
    
    with pdfplumber.open(pdf_path) as pdf:
        # 提取元数据
        result['metadata'] = pdf.metadata
        
        # 逐页提取文字
        for page in pdf.pages:
            page_text = page.extract_text()
            if page_text:
                result['text'] += page_text + "\n"
            
            # 提取表格
            tables = page.extract_tables()
            for table in tables:
                result['tables'].append(table)
    
    return result
```

**适用场景**：
- ✅ 电子 PDF（有文本层）
- ✅ 期刊论文 PDF
- ✅ 课题申报书 PDF
- ✅ 含表格的 PDF

**限制**：
- ❌ 扫描版 PDF（无文本层）需要 OCR
- ❌ 图片中的文字无法直接提取

### 二、Word 排版能力（python-docx）

**已安装库**：python-docx

**支持功能**：
| 功能 | 说明 |
|------|------|
| 文档创建 | 创建新的 Word 文档 |
| 内容写入 | 写入文字、表格、图片 |
| 格式设置 | 字体、字号、颜色、段落格式 |
| 样式应用 | 应用标题样式、正文样式 |
| 表格制作 | 创建表格、设置边框、合并单元格 |
| 页眉页脚 | 设置页眉、页脚、页码 |
| 目录生成 | 自动生成目录 |
| 文档保存 | 保存为.docx 格式 |

**技术实现**：
```python
from docx import Document
from docx.shared import Pt, Cm, RGBColor
from docx.enum.text import WD_ALIGN_PARAGRAPH
from docx.oxml.ns import qn

def create_word_document(content, style_guide='教研论文'):
    """创建格式规范的 Word 文档"""
    doc = Document()
    
    # 页面设置（A4 纸）
    section = doc.sections[0]
    section.page_width = Cm(21)
    section.page_height = Cm(29.7)
    section.top_margin = Cm(3.5)
    section.bottom_margin = Cm(2.5)
    section.left_margin = Cm(2.8)
    section.right_margin = Cm(2.6)
    
    # 添加标题
    if 'title' in content:
        heading = doc.add_heading(content['title'], level=0)
        heading.alignment = WD_ALIGN_PARAGRAPH.CENTER
        heading.runs[0].font.name = '黑体'
        heading.runs[0].font.size = Pt(16)
        heading.runs[0].font.color.rgb = RGBColor(0, 0, 0)
    
    # 添加摘要
    if 'abstract' in content:
        abstract_para = doc.add_paragraph()
        abstract_run = abstract_para.add_run('摘要：' + content['abstract'])
        abstract_run.font.name = '楷体'
        abstract_run.font.size = Pt(12)
    
    # 添加关键词
    if 'keywords' in content:
        kw_para = doc.add_paragraph()
        kw_run = kw_para.add_run('关键词：' + '; '.join(content['keywords']))
        kw_run.font.name = '黑体'
        kw_run.font.size = Pt(12)
    
    # 添加正文
    if 'content' in content:
        for paragraph in content['content']:
            para = doc.add_paragraph()
            para.paragraph_format.line_spacing = 1.5
            para.paragraph_format.first_line_indent = Cm(0.74)
            run = para.add_run(paragraph)
            run.font.name = '宋体'
            run.font.size = Pt(12)
    
    # 添加表格
    if 'tables' in content:
        for table_data in content['tables']:
            table = doc.add_table(rows=len(table_data), cols=len(table_data[0]))
            for i, row in enumerate(table_data):
                for j, cell in enumerate(row):
                    table.cell(i, j).text = str(cell) if cell else ''
    
    # 保存文档
    filename = content.get('filename', 'output.docx')
    doc.save(filename)
    return filename
```

| 省级 | <15% |
| 核心期刊 | <10% |

**原创性第一原则**：教研成果的核心价值在于您真实的教学实践与独立思考。本技能只提供**框架与思路初稿**，最终内容须由您结合自己的课堂原创完成，切勿整段搬用他人成果或 AI 生成文本。

**规范引用建议**（避免无意识重复，也避免学术不端）：
| 做法 | 说明 | 示例 |
|------|------|------|
| 规范标注引用 | 引用他人观点、数据须注明出处，按 GB/T 7714-2015 | 间接引用也要标来源 |
| 区分直接/间接引用 | 原话加引号并注页码，转述亦须标注 | 避免"伪装成自己观点" |
| 以自身实践为主体 | 用本校课例、调查数据、教学反思支撑论点 | 原创比重越大越稳 |
| 写作前先列提纲 | 用自己的逻辑组织内容，而非套用模板 | 降低结构性雷同 |
| 成稿后用正规查重系统自检 | 知网/维普/万方等，以单位指定平台为准 | 先改后交，不留隐患 |

**郑重提醒**：任何"改写降重""同义替换洗稿"都不是本技能的导向。以原创立身，才经得起评审与时间的检验。

### 各部分撰写指导

#### 一、问题的提出

**撰写要点**：
- 从实际教学实践出发
- 描述具体遇到的问题
- 用问题的口气表述（3-4 个）

**示例**：
```
通过多年的初中语文教育教学实践，我发现当前整本书阅读教学中存在以下问题：

1. 如何让学生从"读过"走向"读懂"，形成真实的阅读体验？
2. 如何设计分层任务，兼顾不同阅读基础的学生？
3. 如何将整本书阅读与课标"学习任务群"要求有效对接？
4. 如何建立可操作的整本书阅读过程性评价？
```

#### 二、国内研究状况

**撰写要点**：
- 梳理近 5 年相关研究
- 引用 10-20 篇文献
- 指出研究空白

**示例**：
```
关于初中语文整本书阅读教学策略的研究，国内学者已取得一定成果。

王荣生（2021）在《整本书阅读教学的问题与对策》中提出……

吴欣歆（2022）在《基于学习任务群的整本书阅读设计》中指出……

然而，针对本校学情、可落地的分层任务设计研究仍较薄弱，……
```

#### 三、研究意义

**撰写要点**：
- 理论意义：丰富……理论
- 实践意义：为……提供参考

**示例**：
```
1. 理论意义：丰富初中语文整本书阅读教学策略的理论体系，为"学习任务群"落地提供案例支持。

2. 实践意义：为一线语文教师提供可操作的分层任务与评价工具，提高整本书阅读教学实效。
```

#### 四、理论依据

**撰写要点**：
- 课程标准
- 教育理论
- 学习理论

**示例**：
```
1. 《义务教育语文课程标准（2022 年版）》"整本书阅读"任务群
2. 建构主义学习理论
3. 最近发展区理论
4. 过程性评价理论
```

#### 五、研究内容

**撰写要点**：
- 概念界定清晰
- 研究内容具体
- 用"通过……，总结提炼……"的句式

**示例**：
```
5.1 概念界定

整本书阅读教学策略：指教师为引导学生读通、读懂、读透整本书而设计的目标、任务、活动与评价的总和。

5.2 研究内容

（1）通过文献检索和课标研究，总结提炼整本书阅读的任务群要求及国家对阅读教学的导向。

（2）通过课堂观察与学生问卷，总结提炼当前整本书阅读教学的现状与问题。

（3）通过实践研究，总结提炼分层任务设计、导读课与研读课的教学策略。

（4）通过实践验证，总结提炼整本书阅读过程性评价量表。
```

#### 六、研究方法

**撰写要点**：
- 说明每种方法的具体运用
- 方法与研究内容对应

**示例**：
```
1. 文献研究法：梳理整本书阅读相关理论与课例。
2. 行动研究法：在任教班级持续开展教学实践与反思。
3. 案例研究法：选取典型课例（如《西游记》《骆驼祥子》）深入分析。
4. 调查研究法：通过问卷与访谈了解学生阅读状况。
```

#### 七、研究过程

**撰写要点**：
- 只写做了哪些工作
- 不写研究结论
- 分三个阶段

**示例**：
```
7.1 准备阶段（2024.09-2024.12）
- 研读课标与文献，明确研究问题
- 设计调查问卷与学生阅读基线测试
- 制定研究方案

7.2 研究阶段（2025.01-2025.06）
- 实施分层任务教学，开展导读课、研读课
- 收集课例与学生阅读档案
- 中期总结与调整

7.3 总结阶段（2025.07-2025.09）
- 整理研究资料与数据
- 撰写研究报告
- 形成评价量表与课例集
```

#### 八、研究成果

**撰写要点**：
- 实践方面：学生变化、教师变化
- 理论方面：研究内容扩写
- 用数据支撑

**示例**：
```
（一）实践方面

1. 学生的变化和收获
- 认识：学生从"应付任务"到主动规划阅读，整本书读完率从 60% 提升到 90%
- 能力：能独立撰写读书笔记与人物分析，批注质量明显提升
- 效果：名著阅读题班级平均得分提高约 5 分

2. 教师的变化和收获
- 认识：对"学习任务群"落地有了更具体的抓手
- 能力：分层任务设计与过程性评价能力明显提升
- 效果：形成 3 个可推广的典型课例

（二）理论方面

1. 整本书阅读的任务群要求与教学导向
（1）任务群定位：引导通读、研讨、分享……
（2）教学导向：以学生自主阅读为主……

2. 当前整本书阅读教学的现状与问题
（1）读前：缺乏有效激趣与计划……
（2）读中：任务同质化，忽视差异……

3. 分层任务与课型策略
（1）导读课：
- 激趣导入：……
- 计划发布：……
（2）研读课：
- 任务单驱动：……
- 研讨分享：……

4. 整本书阅读过程性评价量表
（1）评价维度：阅读态度、方法、笔记、分享、成果
（2）评价方式：自评 + 互评 + 教师评
（3）评价标准：分层描述
```

#### 九、研究反思

**撰写要点**：
- 诚恳说明不足
- 明确改进方向

**示例**：
```
1. 研究存在的不足
- 研究样本限于本校任教班级，代表性有待扩大
- 研究时间较短，长期阅读习惯的养成效果有待追踪
- 评价量表还需在更多课型中检验

2. 下一步研究方向
- 扩大研究范围，进行跨校对比研究
- 延长研究周期，关注阅读素养的持续发展
- 完善评价工具，形成标准化量表
```
