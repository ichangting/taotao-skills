# 作文批改（zuowen-grading）

## 目的

为一线语文教师提供**学生作文的结构化批改参考**——把机械批改时间省下来，让教师把精力放在更有价值的面批与教学设计上。

核心理念：**本技能是"批改参考生成器"，不是"自动判分机"。** 所有输出均为建议，教师拥有完全的终审权与教学责任。

## 何时使用

- "帮我批改这篇作文""这篇作文怎么评"
- "给我出个评语""写个鼓励型/严谨型评语"
- "这篇作文主要问题在哪""帮我列个问题清单"
- "帮我做个升格示例""这段怎么改更好"
- "学生作文里的古诗引用对不对"（古文引用审校）
- 日常批改、月考作文、中考模拟作文、竞赛作文辅导。

## 与其他技能的分工（单一数据源）

| 层 | 谁负责 | 说明 |
|---|---|---|
| 语言层审核（语病/错别字/用词/文风） | **doc-review** | 本技能复用 doc-review 四关审核逻辑做"语言关"，不另建规则 |
| 古文/古诗引用核对 | **guwen-cidian** | 学生作文中引用的文言词句、古诗文句，回查 guwen-cidian 验证准确性 |
| 内容/结构/立意/亮点/升格 | **本技能（zuowen-grading）** | 维度评分、夹批、评语、升格示例 |
| 考情对标（可选） | **jiangxi-zhongkao-beikao** | 若已安装，可对标中考评分维度与近年命题趋势 |

> ⚠️ **严禁复制语料**：doc-review 已有的审核清单、guwen-cidian 已有的字词数据，本技能一律"引用"不"拷贝"。

## 核心能力（MVP，五种用法）

### 用法一：全维度批改参考（主场景）

输入一篇学生作文（文本/图片 OCR 后文本），输出一份结构化批改参考报告。格式**锁死**见 `references/批改报告模板.md`，包含：

1. **维度评分建议**：内容（立意/选材）、结构（层次/过渡）、语言（表达/文采）、书写（卷面/规范）四维度，各给参考等级（优/良/中/待提升）+ 1 句依据。
   - ⚠️ 顶部恒带声明：`【AI 建议，教师终审】以下为参考，非最终评定。`
2. **夹批式问题清单**：逐段/逐句标注问题（语病、错别字、逻辑断层、立意偏差、素材不当），每条带"位置 + 问题类型 + 简述 + 修改方向"。
   - 语言层问题复用 doc-review 四关逻辑（文字差错/语法硬伤/用词精准/文风规范）。
3. **亮点标注**：识别并标出学生写得好的句子、构思、选材、修辞，供教师当面肯定。
   - 至少标注 1 处亮点（再弱的作文也有可肯定之处）。
4. **评语草稿**：提供"鼓励型"与"严谨型"两种语气，教师按需取用或混搭。
   - 鼓励型：先肯定 → 再建议 → 收束鼓励。
   - 严谨型：直指核心问题 → 给具体改进路径 → 期待。
5. **升格示例**：针对 1–2 处最典型问题，给出一段改写示范（不替全篇重写），标注"改了什么、为什么这样改"。

### 用法二：单项诊断

教师只需要某一维度的反馈：
- "只看语言问题"→ 仅输出语言层夹批（复用 doc-review）。
- "只看结构"→ 仅输出结构诊断（层次/过渡/详略）。
- "只看立意"→ 仅输出内容/立意分析。

### 用法三：评语生成

教师给关键词或简要印象，生成评语草稿：
- 输入："该生叙事流畅但结尾仓促，选材有新意"
- 输出：鼓励型 + 严谨型两版评语。

### 用法四：升格指导

教师指定某段/某处，生成升格示例 + 改法说明：
- 输入：学生原文片段 + "这里怎么改"
- 输出：改写示范 + 改动理由 + 可迁移的写作技巧点拨。

### 用法五：古文引用审校

学生作文中引用了古诗文，回查 guwen-cidian 验证：
- 引文是否准确（有无错字/漏字/张冠李戴）
- 用法是否恰当（语境是否匹配原意）
- 标注"引用正确""引用有误（附正确版本）""用法可商榷"。

## 输出格式（锁死）

严格按 `references/批改报告模板.md` 渲染。关键约束：

- 报告顶部**必须**有 `【AI 建议，教师终审】` 声明，不可省略。
- 维度评分用"优/良/中/待提升"四级，**不用数字分数**（避免被误读为最终得分）。
- 夹批问题清单按严重程度排序：🔴 硬伤（语病/错字/事实错误）→ 🟡 软伤（逻辑/详略/过渡）→ 🟢 建议（文采/深度/创新）。
- 评语草稿必须提供两种语气版本。
- 升格示例只改 1–2 处，不重写全文。

## 合规红线（不可破）

1. **仅教师端**：本技能不面向学生开放，不做成学生自助批改工具。输出对象是教师，不是学生。
2. **不判分、不排名**：不生成数字分数，不生成全班分数榜/排名表。维度评估用等级（优/良/中/待提升），不用分值。
3. **PII 本地化**：默认本地处理。不存储学生作文到任何外部服务；提醒教师"勿在公开场景粘贴含真实姓名/学校的学生作文"。
4. **不替代教师**：所有输出均为"参考建议"，最终批改、评分、评语的决定权与教学责任在教师。
5. **不向学生收费**：本技能定位免费公共资产，不设计任何付费墙。
6. **尊重学生**：评语措辞尊重学生人格，不讽刺、不贬损；即使"严谨型"也对事不对人。

## 审核原则

- **先肯定后建议**：无论哪种语气，都不全盘否定学生习作。
- **具体可操作**：问题描述具体到句/段，修改方向可操作，不说空话（如"语言需加强"→ 改为"第 3 段连续 4 个'然后'，建议用时间词/动作衔接替换"）。
- **对标课标与考情**：评分维度参照《义务教育语文课程标准（2022 年版）》写作目标；若已安装 jiangxi-zhongkao-beikao，可对标江西中考作文评分维度。
- **语言层复用 doc-review**：语病/错字/用词问题走 doc-review 四关逻辑，保持全库审核标准一致。
- **古文引用回查 guwen-cidian**：学生引用的古诗文句，验证准确性，避免"将错就错"。
