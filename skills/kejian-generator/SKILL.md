# 课件生成（kejian-generator）

## 目的

把教师已有的教案（或课题）转化为**可落地的课件骨架**——幻灯片大纲 + 逐页精简文本 + 板书设计 + 配图建议，补"备课闭环最后一环"。

核心理念：**本技能只转译教案为课件结构，不重写教学逻辑。** 教学设计的判断权在教师；本技能做的是"把教案里的教学流程翻译成幻灯片语言"。

## 何时使用

- "帮我把这份教案做成课件大纲"
- "这节课 PPT 怎么搭""幻灯片大纲"
- "板书设计怎么排""主副板书怎么分"
- "哪些地方需要配图""课件骨架"
- 备课最后一步：教案已定，需要快速搭出课件框架。
- 公开课/赛课前：需要结构清晰、视觉有节奏的课件骨架。

## 与其他技能的分工（单一数据源）

| 层 | 谁负责 | 说明 |
|---|---|---|
| 教学设计（目标/流程/活动/评价） | **k12-lesson-planning-zh** | 本技能消费教案产出，不重新设计教学逻辑 |
| 配图规划与生成 | **article-illustration** | 本技能标注"此处建议配图"，具体配图方案回查 article-illustration |
| 文本审核（课件文字零硬伤） | **doc-review** | 课件中所有文字输出前经 doc-review 收口 |
| 品牌视觉（字体/配色） | **品牌规范** | 标题思源宋体、正文思源黑体、配色石青/杏黄/朱砂 |
| 字词/古文内容准确性 | **guwen-cidian** | 课件涉及文言文字词时回查，确保无误 |

> ⚠️ 本技能不复制教案内容，不复制品牌规范，不复制配图方案——全部"引用"不"拷贝"。

## 核心能力（MVP，四种用法）

### 用法一：教案→课件大纲（主场景）

输入一份教案（文本/JSON/口述课题），输出一份结构化课件大纲。格式**锁死**见 `references/课件结构模板.md`，包含：

1. **幻灯片大纲**：逐页结构，每页标注"页码 + 页面类型 + 标题 + 要点"。
   - 页面类型：封面 / 目标页 / 导入页 / 新知页 / 活动页 / 练习页 / 小结页 / 作业页 / 封底。
   - 每页要点不超过 5 条（课件不是教案，文字必须精简）。
2. **逐页精简文本**：每页的实际展示文字——短句化、关键词醒目、避免大段文字。
   - 铁律：每页正文不超过 60 字（标题除外）。超过则拆页。
3. **板书设计**：主板书（知识结构/核心线索）+ 副板书（临时补充/学生发言记录区）。
   - 给出布局建议（左右分栏/上下分层/中心辐射）。
4. **配图建议**：标注"第 X 页建议配图"，说明配图意图（情境渲染/知识图解/活动指引）。
   - 具体配图方案回查 `article-illustration`（若已安装）。
5. **品牌视觉标注**：标题字体、正文字体、主色/辅色/强调色，回查品牌规范。
6. **文本收口**：所有课件文字经 doc-review 审核，确保零语病、零错字。

### 用法二：单页/局部优化

教师已有课件，需要优化某一页或某几页：
- "这页文字太多，帮我精简"
- "这个环节怎么拆成 2-3 页"
- "板书怎么排更好看"

### 用法三：板书设计专项

教师只需要板书设计：
- 输入课题/教学主线
- 输出：主板书布局 + 副板书建议 + 书写顺序提示

### 用法四：课件节奏诊断

教师已有课件，需要诊断节奏问题：
- "这个课件是不是文字太多了"
- "哪里该加一页活动/互动"
- "导入太长/太短"
- 输出：逐页诊断 + 调整建议（不重写，只标注问题与方向）。

## 输出格式（锁死）

严格按 `references/课件结构模板.md` 渲染。关键约束：

- 输出为 **Markdown 结构化大纲**（或可选 JSON），不直接生成 .pptx。
- 每页必须包含：页码、页面类型、标题、要点（≤5 条）、展示文本（≤60 字）。
- 板书设计必须区分主/副板书。
- 配图建议只标注意图与位置，不内嵌图片。
- 品牌视觉信息在大纲首页统一标注，不逐页重复。

## 课件设计原则

1. **少即是多**：课件是"视觉辅助"，不是"教案投影"。每页只传递 1 个核心信息。
2. **学生视角**：课件是给学生看的，不是给教师自己看的。文字要学生读得懂、跟得上。
3. **节奏感**：文字页与活动页交替，避免连续 5 页以上纯文字。每 3-4 页插入一个互动/活动节点。
4. **留白**：不把页面塞满。宁可多拆一页，不挤在一页里。
5. **一致性**：全课件字体、配色、版式风格统一（回查品牌规范）。
6. **服务教学目标**：每一页都要能回答"这页为哪个教学目标服务"，删掉无目标的装饰页。

## 非目标（明确不做什么）

- **不生成 .pptx 文件**：输出 Markdown/JSON 大纲，留接口给 md2ppt / Marp / reveal.js 或教师手动排版。
- **不替代教学设计**：只转译教案为课件结构，不重写教学目标/流程/活动设计。
- **不生成与统编教材冲突的内容**：课文引用限于个人教学使用，不内置整篇课文原文库。
- **不做动画/交互设计**：只规划内容与结构，不涉及 PPT 动画、触发器、超链接交互。
- **不面向学生**：本技能为教师备课工具，不生成学生端课件。

## 降级策略

- `k12-lesson-planning-zh` 未安装/无教案输入：接受教师口述课题 + 简要流程，按通用课件结构生成大纲。
- `article-illustration` 未安装：配图建议降级为文字描述（"此处建议配一张……的图"），不生成具体配图方案。
- `doc-review` 未安装：文本审核降级为内置基础检查（明显错字/语病），提示教师"完整审核建议安装 doc-review"。

## 合规与版权

- 课文引用限于个人教学使用，公开分发需授权。
- 不内置整篇课文原文库（避免教材版权问题）。
- 配图建议使用公共素材或 AI 生成，不引用受版权保护的商业图片。
- 品牌视觉规范为涛涛个人品牌，使用者可自定义替换。
