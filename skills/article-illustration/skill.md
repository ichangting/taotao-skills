# 文章配图智能规划

## 任务目标
- 本Skill用于:为文章提供深度结构化分析，智能规划配图方案，生成高质量配图
- 能力包含:文章深度分析、配图方案规划、风格类型选择、图像生成、多格式输出
- 触发条件:用户要求配图、设计视觉方案、使用快捷指令(/一键配图、/小红书配图等)

## 前置准备
- 无特殊依赖或前置准备

## 操作步骤

### 步骤1:深度结构化分析文章

首先识别文章类型(故事类/非故事类)。

**非故事类文章分析**:
提取文章核心信息:
- 核心论点与分论点
- 逻辑结构(总分/递进/对比/因果/时间线)
- 关键数据、案例和金句
- 文章用途(公众号、PPT、教案、报告、小红书、技术文档等)
- 目标读者的审美偏好和认知门槛
- 适合视觉表达的逻辑节点

**故事类文章专项分析**:
当识别为故事内容时,执行深度故事分析:
- 主要人物识别(角色、性格、外貌特征、人物关系)
- 情节结构梳理(开端、发展、高潮、结局)
- 环境描写分析(场景、氛围、时间、地点)
- 矛盾冲突定位(人物冲突、环境冲突、内心冲突)
- 关键情节提取(适合视觉表达的情节节点,特别是矛盾冲突的瞬间)

### 步骤2:智能规划配图方案
基于分析结果,规划:
- 配图数量(一般1000字1-2张,3000字3-5张)
- 每张图的类型(从配图类型体系中选择,见references/image-types.md)
- 每张图的风格(从风格体系中选择,见references/style-system.md)
- 每张图的核心表达内容(一句话描述)
- 每张图的建议插入位置

### 步骤3:不确定项提供明确选项
如果以下项无法确定,必须提供2-4个选项:
- 配图数量选项
- 配图类型选项
- 配图风格选项
- 核心表达方向选项

### 步骤4:询问用户具体要求
询问用户是否有具体绘图要求:
- 颜色偏好或禁止颜色
- 构图要求(横版/竖版/正方形、居中/左右布局等)
- 需要包含或禁止的元素
- 尺寸要求(如公众号封面2.35:1、小红书3:4等)
- 其他特殊要求

### 步骤5:校验要求与文章契合度
执行以下校验:
1. 对比文章核心主题、情感基调和使用场景
2. 判断用户要求是否合理、是否存在冲突
3. 若存在冲突:
   - 明确指出具体冲突点
   - 解释冲突原因
   - 给出修正建议
   - 询问用户是否坚持原要求
4. 若无冲突,确认最终方案

### 步骤6:生成最终配图
根据最终确认的方案:
- 为每张图生成详细、高质量的绘图关键词
- 使用平台图像生成能力生成配图
- 生成完成后,询问用户后续操作:
  1. 修改或重绘某一张图
  2. 将所有图片打包成ZIP
  3. 拼接成一张垂直长图
  4. 生成带图片引用的Markdown文件
  5. 其他操作

## 快捷指令处理

当用户输入快捷指令时,直接执行对应工作流,跳过常规流程:

- `/一键配图`:自动完成文章分析→规划→生成全部配图
- `/小红书配图`:将文章拆分为3-9个要点,生成统一风格的小红书系列卡片
- `/公众号配图`:生成1张2.35:1封面图+3张内文图
- `/长图`:将所有配图自动拼接成一张垂直长图
- `/知识漫画`:将文章核心内容转化为分镜式知识漫画
- `/视觉笔记`:将文章干货转化为手绘风格的视觉笔记
- `/故事配图`:自动分析故事的人物、情节、环境、冲突,为关键情节配图
- `/人物设计`:为故事主要人物设计形象插画
- `/情节分镜`:将故事关键情节转化为分镜式插画

## 输出规范

严格遵循以下格式输出配图规划:

**非故事类文章输出格式**:
```
📝 文章分析
核心论点：[文章核心观点]
逻辑结构：[总分 / 递进 / 对比 / 因果 / 时间线]
情感基调：[正式 / 轻松 / 严肃 / 活泼 / 科技 / 文艺]
用途：[公众号 / PPT / 教案 / 报告 / 小红书 / 其他]
目标读者：[目标读者群体]

🎨 配图规划
共推荐 [X] 张配图：
位置：[文章开头 / 第 X 段后 / 结尾]
类型：[配图类型]
风格：[配图风格]
核心内容：[一句话描述这张图要表达的内容]

❓ 需要你确认
[如果有不确定的项,列出选项；如果没有,删除此部分]

💬 补充要求
你是否有具体的绘图要求？例如颜色、构图、元素、尺寸等。如果有,请告诉我。
```

**故事类文章输出格式**:
```
📖 故事分析
故事类型：[小说 / 散文 / 童话 / 寓言 / 传记]
主要人物：[角色名及简要特征]
情节结构：[开端、发展、高潮、结局的简要描述]
环境氛围：[场景、时间、情感氛围]
矛盾冲突：[主要冲突类型和冲突点]
关键情节：[适合配图的关键情节节点]

🎨 配图规划
共推荐 [X] 张配图：
位置：[故事开头 / 第 X 段后 / 关键情节处 / 结尾]
类型：[配图类型]
风格：[故事插画风格]
核心内容：[描述人物/场景/冲突的视觉呈现]

❓ 需要你确认
[如果有不确定的项,列出选项；如果没有,删除此部分]

💬 补充要求
你是否有具体的绘图要求？例如颜色、构图、人物形象、场景细节等。如果有,请告诉我。
```

## 注意事项
- 所有配图类型和风格必须从参考文档提供的体系中选择,不得自行创造
- 严禁模糊回答,不确定项必须提供明确选项供用户选择
- 发现用户要求与文章冲突时,必须主动提醒并给出建议
- 配图的核心目标是强化文章逻辑、提升阅读体验、美观专业、完全贴合内容气质
- 充分利用智能体的图像生成能力,无需编写脚本

## 异常处理规则

### 内容过短
如果用户只发送了一句话或非常简短的内容:
- 说明:"你提供的内容较短,我无法准确分析文章主题和结构。请提供完整的文章内容,或者告诉我你想要生成什么类型、什么内容的配图。"

### 无关内容
如果用户要求生成与文章完全无关的内容:
- 提醒:"你要求生成的内容与文章主题无关,可能会影响文章的连贯性和专业性。建议生成与文章内容相关的配图,你是否坚持原要求？"

### 违规内容
如果用户要求生成违法、违规或不适当的内容:
- 拒绝:"抱歉,我无法生成违法、违规或不适当的内容。请调整你的要求。"

## 使用示例

### 示例1:常规配图流程
用户:"我写了一篇关于AI发展趋势的文章,帮我设计配图方案"

智能体执行:
1. 分析文章内容、逻辑结构、目标读者
2. 规划配图方案(3张图:封面图、信息图、流程图)
3. 提供风格选项(暗黑科技、商务专业、notion简约)
4. 询问用户具体要求
5. 校验并生成配图

### 示例2:使用快捷指令
用户:"/小红书配图"

智能体直接执行:
1. 将文章拆分为5个核心要点
2. 规划生成5张3:4尺寸的小红书卡片
3. 使用马卡龙清新风格
4. 直接生成配图,无需额外询问

### 示例3:冲突校验
用户:"这是一份严肃的技术报告,我想要可爱卡通风格的配图"

智能体执行校验:
- 识别冲突:严肃技术报告 vs 可爱卡通风格
- 主动提醒:"技术报告通常使用专业、简洁的风格,可爱卡通手绘可能会降低文章的专业性。建议使用'notion简约'或'蓝图技术'风格,你是否坚持使用可爱卡通手绘？"

### 示例4:故事配图流程
用户:"这是一个关于小女孩和流浪猫的温馨故事,帮我设计配图"

智能体执行:
1. 识别为故事类内容
2. 分析故事:主要人物(小女孩、流浪猫)、情节结构(相遇→相处→分离→重逢)、环境(城市街道、公园)、矛盾冲突(人与动物的信任冲突)
3. 规划配图方案(4张图:封面图、人物肖像、故事场景、治愈系结局)
4. 提供风格选项(治愈系风格、温暖手绘、水彩文艺)
5. 询问用户关于人物形象、场景细节的具体要求
6. 校验并生成配图
