# 贡献指南

欢迎一线教师、教育研究者、开发者为本仓库贡献技能（Skill）或提改进建议。
下面说清"怎么提"和"一个合格技能长什么样"。

---

## 一、两种贡献方式

| 方式 | 适合 | 怎么做 |
|---|---|---|
| **提 Issue** | 报 bug、要功能、说想法 | 在仓库 `Issues` 里新建，描述清楚场景 |
| **提 Pull Request** | 你已写好一个技能 / 改好了文件 | Fork → 改 → 提 PR，说明改了什么、为什么 |

> 不确定怎么动手？先开一个 Issue 聊，咱们一起定方案。

---

## 二、一个新技能的最小结构

```
skills/<你的技能名>/
├── skill.md          # 必须：技能定义（YAML frontmatter + 正文）
├── README.md         # 建议：是什么、能做什么、触发词、依赖、示例
├── LICENSE          # 若用非 MIT 协议，放这里（默认随仓库 MIT）
├── references/       # 可选：课标、教材、范文等参考资料
├── scripts/         # 可选：渲染 / 处理脚本
└── examples/        # 可选：一个真实输入 + 成品示例
```

### `skill.md` 的 frontmatter（必填字段）

```yaml
---
name: <技能名，全小写、连字符，与文件夹同名>
description: |
  一句话说清"能做什么、什么时候用"。# 这行会被客户端用来判断何时触发，务必写准
version: 1.0.0
author: 你的名字
license: MIT            # 默认 MIT；派生第三方须保留原协议
homepage: https://github.com/ichangting/taotao-skills/tree/main/skills/<技能名>
---
```

正文建议包含：**角色定位 / 触发条件 / 工作流程 / 边界（不做什么）/ 输出格式**。

---

## 三、内容红线（务必遵守）

1. **不收版权文献**：王力、吕叔湘等学者的**原著 PDF / DOCX / 笔记**受版权保护，
   绝不入库。技能里的"思维视角"可基于公开著作**蒸馏推断**，并做免责声明。
2. **不抄教材原文**：课文、课后题等受版权保护内容，**一律由 AI 原创生成**，
   不照抄、不搬运。
3. **不含隐私**：学生数据、内部规划、个人凭据绝不入库。
4. **派生须署名**：若你的技能改自别人（如 k12 派生自 Anthropic），
   目录内放 `LICENSE` + `NOTICE`，保留原协议与作者。

---

## 四、协议与署名

- 本仓库默认 **MIT**（见根 `LICENSE`）。
- 你贡献的内容**版权归你**，发布即表示以 MIT 授权本仓库使用，请保留署名。
- 各技能 `skill.md` 的 `license` 字段以**目录内 `LICENSE` 文件为准**（若有）。

---

## 五、提交前自查

- [ ] 技能文件夹名 = `skill.md` 里的 `name`
- [ ] frontmatter 五个字段齐全（name / description / version / author / license / homepage）
- [ ] `description` 写清了"何时用"，方便客户端准确触发
- [ ] 不含任何版权文献、教材原文、隐私数据
- [ ] 若改自第三方，已放 `LICENSE` + `NOTICE`
- [ ] 至少有一个能跑通的示例（输入 + 成品）

满足以上，开 PR 即可。谢谢你的分享，让更多老师用上更好的 AI 教学助手。
