# D 方案：缺口技能规划（古诗文鉴赏 / 作文批改 / 课件生成）

> **状态**：✅ 已完成 —— 三个技能全部落地。`gushi-appreciation`（古诗文鉴赏）2026-07-20 v1.0.0；`zuowen-grading`（作文批改）2026-07-23 v1.0.0；`kejian-generator`（课件生成）2026-07-23 v1.0.0。
> **对齐**：`ARCHITECTURE.md` §2/§3/§7 — 命名规范 `<动词>-<对象>`、四域 category、depends-on 数据流、师德红线
> **第一性原理**：回归"教师到底在做什么"——备 → 教 → 练/批 → 评 → 研 → 传播 → 成长。当前三个真空分别落在 **文言深度教学（教）、作文反馈（练/批）、备课闭环收口（备）**。

---

## 0. 规划原则

1. **补环不重复**：三技能都补现有 12 技能没覆盖的环节，不抢 guwen-cidian（只管字词）、doc-review（只审教师写作）、k12（只出教案）的活。
2. **单一数据源**：均回查 `guwen-cidian` 词库 + L2 模板库/品牌规范，禁止复制语料。
3. **护栏内建**：作文评语、课件文本、鉴赏文字默认经 `doc-review` 收口（古文引用专项复用 guwen 数据）。
4. **红线不可破**：创始人在职公办教师 —— **不向学生收费、不替代教师判分、学生数据本地化、不设计任何付费墙**。所有技能定位免费公共资产。

## 2. 逐个详规

### 2.1 古诗文鉴赏 `gushi-appreciation`（category: 文言古汉语）

**定位**：为语文教师提供"单篇古诗文"的鉴赏教学支持——从字词（guwen 已解决）往上走到意象、手法、情感、考法，而非给学生标准答案。

**用户故事**
> 作为【初中语文教师】，我希望【输入一篇古诗文就能拿到结构化的鉴赏素材】，以便【备课/公开课时快速组织赏析环节，且能区分"主流定评"与"可讨论点"】。

**核心能力（MVP）**
1. 单篇鉴赏卡：原文 + 主旨 + 意象分析 + 艺术手法（借景抒情/托物言志/用典/虚实…） + 名句赏析 + 背诵默写易错字（回查 guwen 通假/易错字表）。
2. 手法库：按年级/文体归类的常见鉴赏术语与判定线索。
3. 意象词典：月/柳/酒/雁/落花等高频意象的常规含义与反例。
4. 中考链接：该篇在近年江西/全国中考的考查频次与题型（回查 `jiangxi-zhongkao-beikao` / `wordfreq` 数据）。
5. 定评 vs 商榷：标注"学界主流定评"与"教学可展开的讨论点"，避免唯一标准答案。
6. 输出经 `doc-review` 古文引用专项收口（作者/篇目/原文核对）。

**非目标（明确不做什么）**
- 不替代学生背诵与独立思考。
- 不生成"标准答案唯一版"鉴赏（语文鉴赏本就多元）。
- 不评学生鉴赏题得分。

**成功指标（上线后衡量）**
- 北极星：教师单次鉴赏备课耗时下降比例（目标 ≥40%）。
- 驱动：单篇鉴赏卡被 `k12` / 整本书阅读 引用次数；guwen 词库回查命中率。

---

### 2.2 作文批改 `zuowen-grading`（category: 评价考情）⚠️ 合规最敏感

**定位**：**教师辅助批改参考生成器**——帮教师提高批改效率、统一尺度，但**最终判定权与责任永远在教师**。

**用户故事**
> 作为【语文教师】，我希望【把学生作文交给技能后拿到维度评分建议 + 问题清单 + 评语草稿 + 升格示例】，以便【把机械批改时间省下来做更有价值的面批与教学设计】。

**核心能力（MVP）**
1. 维度评分建议：内容/结构/语言/书写 四维度参考分（**明确标注"非最终分，教师终审"**）。
2. 问题清单（夹批式）：语病、错别字、逻辑断层、立意偏差——复用 `doc-review` 四关审核逻辑做语言层。
3. 评语草稿：提供"鼓励型 / 严谨型"两种语气，教师按需取用。
4. 升格示例：针对 1–2 处问题给出一段改写示范（不替全篇重写）。
5. 亮点标注：识别并标出学生写得好的句子/构思，供教师当面肯定。
6. 古文引用审校：学生作文里的文言/古诗引用回查 `guwen-cidian`。

**非目标（红线相关，强制）**
- **不给学生端使用**（不面向学生开放、不做成学生自助批改）。
- **不自动出分并排名**（不生成全班分数榜）。
- **不存储学生作文到任何外部服务**（默认本地处理，姓名/学校等 PII 不落库、不上传）。
- 不替代教师判分与教学责任。

**成功指标**
- 北极星：教师单篇批改耗时下降比例（目标 ≥50%）。
- 驱动：生成的评语草稿被教师"直接采用率"；问题清单与教师最终批改的一致率。
- 健康：零学生 PII 外传事件（合规硬指标）。

---

### 2.3 课件生成 `kejian-generator`（category: 备课设计）

**定位**：把 `k12-lesson-planning-zh` 产出的教案，转成可落地的课件大纲与逐页文本——补"备课闭环最后一环"。

**用户故事**
> 作为【语文教师】，我希望【给一份教案或课题，就能拿到幻灯片大纲 + 逐页精简文本 + 板书设计 + 配图建议】，以便【直接进 PPT 工具排版，不必从零搭骨架】。

**核心能力（MVP）**
1. 幻灯片大纲：逐页结构（目标 / 导入 / 新知 / 活动 / 小结 / 作业）。
2. 逐页精简文本：短句化、重点词醒目（避免课件文字过密）。
3. 板书设计：主副板书布局建议。
4. 配图建议：回查 `article-illustration` 规划配图点，呼应品牌视觉。
5. 品牌视觉：标题用思源宋体、配色石青/杏黄（回查品牌规范）。
6. 文本经 `doc-review` 收口（课件文字零硬伤）。
7. 输出格式：Markdown / JSON 大纲（**不直接生成 .pptx**，避免绑定 Office，留接口接 md2mp 类工具）。

**非目标**
- 不自动排版成 .pptx（留作后续接 md2mp / 云PPT 接口）。
- 不替代教师的教学设计判断（只转译教案，不重写教学逻辑）。
- 不生成与统编教材冲突的内容。

**成功指标**
- 北极星：教案→课件骨架的搭建耗时下降比例（目标 ≥60%）。
- 驱动：生成的课件大纲被教师"基本不改直接用的比例"。

## 3. 优先级与落地顺序（RICE，规划假设值待数据校准）

| 技能 | Reach | Impact | Confidence | Effort(人周) | RICE | 排序 |
|---|---|---|---|---|---|---|
| 作文批改 | 高（全体语文教师） | 1.0 | 0.7 | 3 | **最高** | ① |
| 古诗文鉴赏 | 中（教古诗文教师） | 0.8 | 0.7 | 2 | 中高 | ② |
| 课件生成 | 高（全体） | 0.6 | 0.6 | 4 | 中 | ③ |

**推荐落地顺序**
1. **古诗文鉴赏** 先行：低成本（复用 guwen 数据）、低风险，可快速验证"中枢数据被新技能消费"的编排范式是否成立。
2. **作文批改** 次之：价值最高，但**必须先把手写合规护栏设计扎实**（仅教师端、不判分、PII 本地化）再动手。
3. **课件生成** 最后：effort 最高且外部竞品多（各类 PPT AI），差异化靠"紧扣新课标+江西中考+品牌一致"，宜在教案上游稳定后再做。

## 4. 跨技能编排示意（depends-on 数据流）

```
k12-lesson-planning-zh ──┐
                         ├─→ kejian-generator ──→ doc-review(收口)
article-illustration ───┘
guwen-cidian ──→ gushi-appreciation ──→ doc-review(古文专项)
doc-review + guwen-cidian ──→ zuowen-grading
```

所有 L1 生产类技能 ← L2（guwen 词库 / 品牌规范 / 模板库 / 教材版本）。

## 5. 风险与护栏（红线清单）

| 风险点 | 对应技能 | 护栏设计 |
|---|---|---|
| 向学生收费 / 有偿补课 | 全部（尤其作文批改） | 技能说明明确"免费公共资产、仅教师端"；不设计付费墙；与小程序/站点无支付模块一致 |
| 替代教师判分、责任转移 | 作文批改 | 输出恒带"AI 建议，教师终审"；不自动出分排名 |
| 学生 PII 外传 | 作文批改 | 默认本地处理；不落库不上传；文档声明"勿粘贴含敏感信息的真实姓名/学校" |
| 教材版权 | 课件生成 | 引用课文限于个人教学使用；公开分发需授权；不内置整篇课文原文库 |
| 鉴赏唯一标准答案 | 古诗文鉴赏 | 标注"主流定评 vs 可讨论"；不替代学生思考 |
| 品牌不一致 | 课件生成 | 强制回查品牌规范（字体/配色） |
