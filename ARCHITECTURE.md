# 涛涛 Skills 体系架构（ARCHITECTURE）

> 本文件定义 taotao-skills 的整体架构与演化方向，供贡献者与使用者理解"技能为什么这样组织"。
> 配套文档：README（按用户场景的导览）、docs/技能体系蓝图.md（第一性原理推导过程）。

## 1. 设计原则（第一性原理）

回归"教师到底在做什么"，而非"我有什么工具"：

1. **以工作流闭环为主轴**：备 → 教 → 练/批 → 评 → 研 → 输出/传播 → 成长。技能按它所处的环节组织。
2. **用户分层**：新手 / 熟手 / 中考毕业班 / 文言专长 / 教研命题 / 学生（规划）。同一工具箱，不同人不同入口。
3. **单一数据源**：文言文词库等共享知识只存一份（guwen-cidian references），其余引用，禁止复制。
4. **格式锁死**：生产类技能输出格式由模板/脚本固定，不靠模型临场发挥。
5. **护栏内建**：关键产出（教案/论文/公众号文）默认经 doc-review 收口。

## 2. 三层架构

### L0 编排层（Orchestration）
- `teacher-workflow`（规划中元技能）：识别意图 → 路由技能链 → 串联数据流 → 产出收口。
- 场景包（组合预设）：备课包 / 中考包 / 整本书包 / 论文包 / 学生包（规划）。

### L1 能力层（Capabilities）— 按工作流环节分 4 域
| 域 | 现有技能 | 规划中 |
|---|---|---|
| 备课设计 | 备课助手(k12)、整本书阅读、跨学科设计、配图规划 | 课件生成 |
| 文言古汉语（中枢） | 文言词典(guwen-cidian)、王力视角、吕叔湘视角、古诗文鉴赏(gushi-appreciation) | — |
| 评价考情 | 江西中考备考、中考分析(通用)、文本词频 | 作文批改 |
| 教研写作（含护栏） | 教研写作、文稿审核(doc-review) | 学生自学卡 |

### L2 数据底座（Foundation）— 单一数据源
- 文言词库（guwen-cidian/references，全库中枢）
- 品牌声音规范（docs/品牌声音与品牌规范.md）
- 通用模板库（教案/论文/报告/审核卡）
- 教材版本库

## 3. 命名规范

- 英文 id：`<动词>-<对象>`（如 guwen-cidian、doc-review、wordfreq）。
- 中文显示名：frontmatter 加 `display-name` 字段（如 文言词典、文稿审核），README 与入口用中文。
- 专家型视角技能（王力/吕叔湘）保留人物名，因定位即"某学人思维"。

## 4. frontmatter 约定（新增字段）

每个 SKILL.md 建议含：

```
name: <英文id>
display-name: <中文显示名>
description: <一句话>
version: <语义化版本>
license: MIT|Apache-2.0
category: 备课设计|文言古汉语|评价考情|教研写作
depends-on: [guwen-cidian]   # 可选，声明数据/能力依赖
```

`category` 用于分层路由；`depends-on` 让编排层自动串链，且改名不破链。

## 5. 依赖与数据流

- `doc-review` → `guwen-cidian`：审核古文引用时回查词库（轻量）。
- `古诗文鉴赏(gushi-appreciation)` → `guwen-cidian`：字词/通假/默写易错字回查词库；→ `doc-review`：古文引用专项收口（作者/篇目/原文核对）。
- `备课助手(k12)` 备文言/古诗词时 → 调 `guwen-cidian` 做字词与串讲。
- 所有 L1 生产类技能 → L2 取模板 / 品牌规范 / 词库。
- 关键产出 → `doc-review` 收口（护栏内建）。

## 6. 用户分层入口矩阵

| 用户 | 推荐起点 |
|---|---|
| 新手教师 | 备课助手 + 整本书阅读 + 文稿审核 |
| 中考毕业班 | 江西中考备考 + 中考分析 + 整本书阅读 |
| 文言专长 | 文言词典 + 王力/吕叔湘视角 + 文稿审核 |
| 教研命题 | 教研写作 + 文稿审核 + 中考分析 |
| 学生(规划) | 文言自学卡 + 作文互评 |

## 7. 缺口与 v2 路线图

| 缺口 | 解决什么 | 优先级 | 状态 |
|---|---|---|---|
| 古诗文鉴赏 | 古诗词/文言文鉴赏教学（guwen 只管字词） | 高 | ✅ 已实现（gushi-appreciation v1.0.0，2026-07-20） |
| 作文批改 | 学生作文批改反馈（仅教师端、不判分、PII 本地化） | 高 | 待实现 |
| 课件生成 | 备课闭环最后一环 | 中 | 待实现 |
| 课堂活动设计 | 补"教"的真空 | 中 | 待实现 |
| 学生自学卡 | 学生端自主学习闭环 | 低（合规谨慎） | 待实现 |

合规：学生端须规避收费/有偿补课红线，沿用免费公共资产定位。

## 8. 演进方向

- 落地 `teacher-workflow` 元技能，让"我说要备一节课"自动编排 k12 + guwen + perspective + article + doc-review。
- 场景包以"组合预设"形式写入 README 与元技能，降低新手门槛。
